# dev-debug-panel audit — 2026-05-18

## State of play

A 24 KB single-class `DebugPanel` that injects a draggable/resizable DOM
overlay. Has `debug()`, `log()`, `debugState()`, a config tab for
namespace toggles wired via `setLoggerApi(...)`, and a global `debug()`
function that goes through `eventbusjs` to a panel created elsewhere.

## Concrete problems found

| # | Where | Problem |
|---|-------|---------|
| 1 | `DebugPanel` constructor | Auto-appends to `document.body` — no way to mount to a custom root, no way to *not* mount yet. Framework integrations don't get a clean lifecycle. |
| 2 | `debug()` function | Uses `eventbusjs` to dispatch a global event. That works only if a `DebugPanel` was already constructed; otherwise the event is silently dropped. Also drags in eventbusjs as a peer dependency for a single-listener fanout. |
| 3 | Tabs | One tab per namespace. The user's spec says one tab per `id` (different concept). Currently logs from different ids in the same namespace pile into one tab; there's no per-id grouping. |
| 4 | Tab controls | Only the *current* tab has a "Clear" button (in the global toolbar). Per-tab "Copy" is per-entry only, not whole-tab. The user wants tab-level "Copy" + "Clear" icons, consistent. |
| 5 | Keyboard shortcut | `Ctrl+Alt+D`. The user wants `Shift+Alt+D` (Chrome reserves Ctrl+Alt+D for nothing in particular but it conflicts with macOS desktop). |
| 6 | `setLoggerApi(...)` | Imperative wiring after construction. Should be a constructor option. |
| 7 | Coupling to dev-loggers | The panel hand-rolls its own `LogEvent` type and ignores the one in dev-loggers, even though the panel *is* a sink. Should implement `Sink` directly. |
| 8 | `debug(idOrState: any, state?: any)` | First param accepted as `any` makes type inference useless. Tighten to `(id: string, ...args)`. |
| 9 | Build | Outputs `.umd.js` as ESM (see `build.js` line 79 — `format: 'esm'` under the UMD branch). Misleading; either fix the format or drop UMD. |
| 10 | SCSS | Lots of styles for narrow-panel responsive behavior — kept. The "global" tab toolbar exists but per-tab toolbars don't. |

## Improvements (this PR — v2)

- **`debug(id, ...args)` writes via the dev-loggers `debug()` API** when
  available; otherwise the panel registers itself as a global sink and
  the function still works standalone. No more eventbusjs.
- **Per-id tabs**: log entries are bucketed by `id`. The `global` tab
  shows everything chronologically (this part is unchanged).
- **Per-tab toolbar** with Copy + Clear icons, consistent across all tabs.
  Global toolbar keeps Opacity + Hide.
- **Constructor option `loggers`** — pass the `dev-loggers` API once
  at construction; no `setLoggerApi` later. Old method kept for back-compat.
- **Generic mounting** — `new DebugPanel({ mount: false })` produces an
  unmounted panel whose `.element` can be appended anywhere.
  Convenience: `mountPreact(panel, parent?)` registers the panel inside
  a Preact tree as an effect (no `useEffect` boilerplate at every call
  site). Same for `mountReact()` later if needed.
- **`Shift+Alt+D`** as the new shortcut. Constructor option `shortcut`
  lets the host override.
- **Auto-attach to dev-loggers** when present: `panel.attachToLoggers()`
  is a one-liner that resolves the singleton registry and registers the
  panel as a `Sink`.
- **Drop eventbusjs**. The library no longer depends on it. `safe-stringify`
  + `jsondiffpatch` are kept (genuine value).

## Migration delta (v1 → v2)

- `new DebugPanel({ position, width, height })` — same.
- `new DebugPanel({ loggers })` — new optional arg; pass the dev-loggers
  module for automatic config-tab wiring + sink registration.
- `panel.setLoggerApi(...)` still works but is deprecated.
- `debug('id', obj)` no longer requires a `DebugPanel` instance to
  exist before the call — but the call still emits a `LogEvent` via
  dev-loggers, so importing dev-loggers is a hard prerequisite (was a
  soft one before).
- `Ctrl+Alt+D` → `Shift+Alt+D` (rebindable).
- `addLogModule(...)` integration pattern from the README is replaced by
  `loggers.attachSink(panel)` or the `loggers` constructor option.

## What stays the same

- The dragging / resizing / opacity slider / settings persistence flow.
- `JsonView` and the diff-based update behavior.
- The `Settings → Config` tab UI for per-namespace toggles.
- Public method names: `show()`, `hide()`, `toggle()`, `log()`, `debug()`,
  `debugState()`.
